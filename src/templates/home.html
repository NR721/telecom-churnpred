TrustTelecom Churn Prediction

Every month, thousands of customers leave service providers; acquiring new customers is 5 times more expensive than retaining existing ones. As such, our data engineering team created a churn prediction tool (trained on a dataset of 7043 customers based in California), empowering the team to identify high-risk customers, provide targeted offers and discounts, and improve overall customer satisfaction.

Methodology

Data Collection & Cleaning

The dataset used in this model consists of 7,043 customer records from California, containing customer demographics, account details, billing history, and service subscriptions. Before training the model, the dataset was cleaned and preprocessed, which included handling missing values, encoding categorical variables, and normalizing numerical fields.

Feature Engineering

The dataset included both numerical and categorical features. To improve model accuracy, we performed:

  • One-hot encoding for categorical variables (e.g., contract type).
  • Normalization for numerical features (e.g., total charges per quarter).
  • Feature selection to identify key drivers of churn.

Model Selection & Training

Several machine learning models were evaluated, including:

  • Logistic Regression
  • Random Forest
  • XGBoost
The models were assessed using the AUC (Area Under Curve) metric. The Random Forest Classifier achieved the highest AUC score and was selected for deployment.

Model Deployment

The final model was deployed using a Flask API, allowing users to submit customer data and receive real-time churn predictions. The API was containerized using Docker for scalability.
